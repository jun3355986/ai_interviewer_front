# 提案：集成 Spring Cloud 后端服务

## 概述
将 AI 面试官助手前端从静态模拟数据切换为完整的 Spring Cloud 微服务架构集成。这包括实现身份验证、职位管理和匹配分析功能。

## 问题陈述
当前的 Flutter 前端 (`ai_interviewer_front`) 使用静态数据进行 UI 展示。为了提供实际功能，它需要与运行在 9001 端口的用户服务和 9004 端口的职位服务进行通信。

## 提议方案
1. **基础设施**: 引入 `dio` 处理网络库，使用 `shared_preferences` 进行本地存储（Token 管理）。
2. **架构设计**: 实现清晰的服务导向架构：
    - `lib/api/`: 针对不同服务的 Dio 客户端。
    - `lib/models/`: JSON 序列化的数据模型。
    - `lib/services/`: 抽象 API 调用的业务服务层。
3. **核心功能对接**:
    - **身份验证**: 登录、注册和 JWT Token 管理。
    - **职位管理**: 职位列表获取、搜索和详情查看。
    - **匹配分析**: 集成职位与简历的匹配分析 API。
4. **UI 集成**: 更新现有页面（`LoginPage`、`HomePage`、`UploadResumePage`等）以使用新服务。

## 范围
- 用户服务集成（登录、注册、个人资料、更新、登出、Token 刷新）。
- 职位服务集成（职位列表、详情、发布、更新、删除、搜索、匹配分析、我的发布）。
- 完整覆盖 API 文档中的所有接口。
- 为所有网络操作添加错误处理和加载状态。

## 约束
- 后端服务运行在本地端口 9001 和 9004。
- 使用 `dio` 库及其拦截器功能，实现 JWT 自动注入和 401 处理。
- 尽量减少对现有 UI 布局的更改，专注于逻辑对接。
